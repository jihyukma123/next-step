# 엔티티 목록

## 1. 회원 (User)

- **설명**: 시스템을 이용하는 개별 사용자
- **엔티티 속성**:
  - `user_id` (PK): 고유 식별자
  - `nickname`: 사용자 닉네임 (중복 허용 여부 결정 필요)
  - `email`: 계정 이메일 (Unique)
  - `created_at`: 가입 일시
  - `last_login_at`: 마지막 접속 일시

마지막 접속 일시는 왜 필요한지 궁금해서 제미나이한테 물어봄

- 휴면 계정 처리 및 보안 관리(법적/운영적 근거) -> 지금 시점에서는 제일 설득되는 데이터임.
  - 법적 규제: 많은 국가의 개인정보 보호법에 따라, 일정 기간 접속하지 않은 사용자의 데이터를 분리 보관하거나 파기해야 함. 마지막으로 접속한 시점의 데이터가 없으면 이를 처리할 방법이 없음.
  - 계정 도용 감지..? -> 이거는 약간 공감하기 어려운게, 평소 접속하던 주기랑 동떨어진 접속 탐지를 위해서라는데 공감되지 않음, 장소면 모를까.
- 학습 공백기 판단 -> 로그인 시저을 기준으로 다시 로그인 했을 때 학습 공백기를 판단하는 용도로 사용한다는데....이것도 공감되지 않음. 왜냐하면 내 생각에는 이런 서비스는 로그인 상태를 유지할 방법을 찾아서 유지시켜주는게 사용자 편의성이 증가할 것 같은데...(근데 그러면 세션을 어떻게 분리하지? 그것에 대해서도 생각이 필요할 것 같은데 일단은...고..)
- 리소스 낭비 방지: 만약에 배치성으로 돌아가는 작업이 있다면, 접속을 일정 기간 이상 하지 않은 사용자의 경우 이런 배치성 작업을 비활성화해서 리소스 낭비를 방지할 수 있음.(이거는 지금 단계에서 고려할 문제는 전혀 아니라고 판단됨.)

## 2. 학습 도메인 (Domain)

- **설명**: 최상위 학습 카테고리 (지식의 대분류)
- **엔티티 속성**:
  - `domain_id` (PK): 도메인 고유 식별자
  - `user_id` (FK): 생성한 회원 ID
  - `name`: 도메인 이름 (ex. 백엔드 개발)
  - `description`: 도메인 상세 설명 (공부 이유 등)
  - `created_at`: 생성 일시

## 3. 서브도메인 (Sub-Domain)

- **설명**: 도메인 내 세부 과목 (상태 관리 및 우선순위의 단위)
- **엔티티 속성**:
  - `sub_domain_id` (PK): 서브도메인 고유 식별자
  - `domain_id` (FK): 소속된 도메인 ID
  - `name`: 서브도메인 이름 (ex. Java, Spring)
  - `goal_description`: 구체적 목표 설명 (AI 분석의 소스)
  - `priority`: 우선순위 (1, 2, 3 등급)
  - `status`: 진행 상태 (진행 중 / 정체 / 완료)
  - `last_studied_at`: 해당 서브도메인을 마지막으로 학습한 날짜

last_studied_at -> 이거는 왜 필요할지?

- 마지막으로 공부했던 시점을 기록할 방법이 있으면, 차후 복습 대상 여부 판별 시 사용할 수 있을 듯?
- 그리고 추천 알고리즘 구성시에도, 오랫동안 공부하지 않았다고 했을 때 공부할 주제로 만들어줄 수 있을 것 같음.
- 정렬 기능 제공 시, 정렬의 기준이 될 수 있음.(가장 최근에 공부한 순 등등...)

다만 이걸 업데이트할 기준이 있어야할 것 같음.

## 4. 학습 토픽 (Topic)

- **설명**: 서브도메인 아래에서 실행되는 하루 단위의 구체적 주제
- **엔티티 속성**:
  - `topic_id` (PK): 토픽 고유 식별자
  - `sub_domain_id` (FK): 소속된 서브도메인 ID
  - `title`: 공부할 주제명
  - `is_ai_recommended`: AI 추천 여부 (Boolean)
  - `status`: 완료 여부 (진행 중 / 완료)
  - `created_at`: 생성 일시

## 5. 공부 세션 (Study Session)

- **설명**: 특정 토픽을 공부한 실제 시간 기록 (통계용)
- **엔티티 속성**:
  - `session_id` (PK): 세션 고유 식별자
  - `topic_id` (FK): 해당 학습 세션이 속한 토픽 ID
  - `start_at`: 공부 시작 일시 (Timestamp)
  - `end_at`: 공부 종료 일시 (Timestamp)
  - `session_status`: 세션 상태 (진행 중 / 정상 종료 / 강제 종료)
  - _Note_: 세션 종료 시 `end_at - start_at`을 통해 순수 공부 시간 계산 가능.

## 6. 공부 기록 (Log)

- **설명**: 토픽에 대한 학습 결과물 (마크다운 텍스트)
- **엔티티 속성**:
  - `log_id` (PK): 로그 고유 식별자
  - `topic_id` (FK, Unique): 소속된 토픽 ID (1:1 관계를 위해 Unique 제약)
  - `content`: 학습 내용 (Markdown Text / Long Text)
  - `updated_at`: 최종 수정 일시

## 7. 전반적인 진척도 (Overall Status)

- **설명**: AI가 도메인별 학습 데이터를 분석하여 생성한 요약 리포트
- **엔티티 속성**:
  - `status_id` (PK): 요약 정보 고유 식별자
  - `domain_id` (FK): 대상 도메인 ID
  - `report_date`: 리포트 생성 날짜 (Daily Snapshot)
  - `ai_summary`: AI가 요약한 현재 상태 및 평가
