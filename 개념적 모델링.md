# 개념적 모델링 v2

README의 프로젝트 주제를 간소화한 결과에 따라, 개념적 모델링을 다시 해보고자 생성된 문서.

이전에는 요구사항을 별도 문서로 작성했는데, 개념적 모델링이 요구사항에서부터 시작해야되기 때문에 `사용자 스토리` - `요구사항`을 같은 문서에 작성함.

# 사용자 스토리

- 학습자로서, 책상에 앉았을 때 고민없이 오늘 학습할 주제를 빠르게 선택하고 시작하고 싶다. 시간과 에너지를 아끼고 집중하기 위해서.
- 학습자로서, 특정 주제에 대해서 학습했던 히스토리에 대해서 overview를 제공받고 싶다.
- 학습자로서, 내가 관심있는 분야와 관련해서 최근에 학습했던 내용이 오늘의 학습 주제를 결정하는데 반영되면 좋겠다.

# 요구사항

개념적 모델링을 진행하기 위해서 문장 단위 요구사항 작성

(우선순위를 P0/P1 등으로 표기, P0가 우선순위가 가장 높음)

우선순위는 구현 중요도이고, 결국 핵심 아이디어 동작 검증이 안되면 나머지는 의미가 없음.

## [P1] 학습 주제 선정

- 사용자가 저장한 관심 분야와, 최근 학습 기록 및 누적 학습 요약 정보를 활용해서 주제를 추천할 수 있어야 한다.
- 3가지 주제를 추천해서, 사용자가 한 가지를 선택할 수 있어야 한다.

## [P2] 학습 기록 관리

- 사용자가 어떤 주제를 언제 학습했는지 기록되어야 한다.
- 사용자가 추천받은 주제를 학습한 이후에, 학습한 내용을 간략하게 입력해서 저장할 수 있어야 한다.
- 사용자가 학습한 내용을 제거할 수 있어야 한다.
- 사용자가 학습한 내용을 재학습하고, 작성했던 간략 정리를 업데이트 할 수 있어야 한다.

## [P3] 누적 학습 요약

- 사용자가 저장한 도메인/서브 도메인별로, 누적 학습 내용을 요약한 데이터가 학습 기록 추가 시 업데이트 되어야 한다.

## [P4] 사용자 관심 분야 관리(학습하고자 하는 주제)

- 사용자는 학습하고자 하는 관심 분야를 지정할 수 있어야 한다.
- 사용자는 관심 분야를 도메인 - 서브도메인 순으로 정리할 수 있어야 한다.
- 사용자는 관심 분야에 대해서 공부 목적이나, 범위 등에 대한 간략한 설명을 추가할 수 있어야 한다.
- 사용자는 도메인/서브 도메인을 삭제/업데이트 할 수 있어야 한다.
  - 관심 분야 삭제 시 관련 학습 기록은 유지되며, 분야 참조는 보존된다 (Soft Delete)
- 사용자는 상세 설명을 업데이트 할 수 있어야 한다.

### 삭제 정책

- 관심 분야 삭제 시 관련 학습 기록은 보존된다.
- 삭제된 관심 분야에 학습 기록이 있는 경우 경고 메세지를 표시한다.
- 관심 분야는 논리적 삭제되어 복구 가능하다.

## [P5] 사용자 관리

- 사용자는 소셜 로그인(Google)을 활용해서 회원가입/로그인할 수 있어야 한다.
- 사용자는 로그아웃할 수 있어야 한다.
- 사용자 세션은 7일 동안 유지되어야 한다.

## 명사 동사 추출

- 관심분야: 도메인 - 서브 도메인

동사

- 학습 주제를 추천한다
- 학습한 주제를 간략하게 기록한다
- 관심분야에 대한 설명을 추가한다
- 관심분야를 사용자가 삭제했을 때 실제로는 DB에 soft delete형태로 유지되어야 한다.
- 사용자는 소셜 로그인을 통해서 로그인한다
- 학습 내용이 누적되어서 업데이트 되어야 한다.

- 누적 학습 요약
