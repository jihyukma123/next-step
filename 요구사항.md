# 📑 최종 학습 관리 시스템 요구사항 명세서

## 1. 회원 (User)

- **회원**은 가입 시 고유 식별값, 닉네임, 생성일 정보를 가진다.
- **회원**은 본인의 학습 체계를 위해 여러 개의 **도메인(Domain)**을 관리한다.

## 2. 학습 도메인 (Domain)

- **도메인**은 최상위 학습 분류이다. (ex. 백엔드, 주식)
- **도메인**은 이름과 상세 설명을 가지며, 하위에 여러 **서브도메인**을 가진다. (1:N)
- **AI**는 도메인 단위로 학습 데이터를 분석하여 **진척도 요약(Overall Status)**을 제공한다.

## 3. 서브도메인 (Sub-Domain)

- **서브도메인**은 도메인 내의 구체적인 학습 과목이다. (ex. Java, Spring)
- 이름, 목표 설명, 우선순위, 현재 상태(진행 중/완료)를 관리한다.
- 하나의 서브도메인은 여러 개의 **토픽(Topic)**을 가질 수 있다. (1:N)

## 4. 학습 토픽 (Topic)

- **토픽**은 서브도메인 내에서 실제로 실행되는 구체적인 공부 주제이다. (ex. Java의 동시성 제어)
- **토픽**은 AI 추천 혹은 사용자 직접 입력을 통해 생성된다.
- **토픽**은 하나의 통합된 **공부 기록(Log)**을 가진다. (1:1 관계)
- **토픽**은 자신의 학습을 위해 발생한 여러 개의 **공부 세션**을 가진다. (1:N 관계). 한 번의 세션 동안 해당 토픽을 마무리하지 못하면, 다음 세션에서 이어서 해당 토픽을 공부할 수 있음.
- **상태 관리**: 사용자가 명시적으로 '학습 완료'를 선택하기 전까지는 '진행 중' 상태를 유지하며, 이 기간 동안 기록은 계속 누적/수정된다.

## 5. 공부 세션 (Study Session)

- **세션**은 사용자가 특정 토픽을 공부하기 위해 '학습 시작'을 누를 때마다 생성되는 **시간 기록 단위**이다.
- **속성**: 학습 날짜, 시작 시간, 종료 시간, 총 학습 시간, 세션 상태(진행 중/종료).
- **시나리오**: 사용자가 공부를 중단했다가 나중에 다시 시작하면, 새로운 세션이 생성되지만 기존 토픽과 로그를 그대로 불러온다.
- **통계**: 세션 데이터는 사용자가 '언제, 얼마나' 공부했는지 측정하는 통계 용도로만 사용된다.

## 6. 공부 기록 (Log)

- **기록**은 특정 **토픽**에 종속된 마크다운 형식의 텍스트 데이터이다.
- **누적 업데이트**: 새로운 세션이 시작될 때마다 기존 로그 내용을 불러와서 편집(Update)하는 방식으로 관리된다.
- 하나의 토픽에 대해 오직 하나의 로그만 존재하며, 토픽이 완료되면 최종 저장된다.

## 7. 전반적인 진척도 (Overall Status)

- **AI**가 **도메인**별로 축적된 **로그(Log)**들을 분석하여 생성하는 요약 정보이다.
- **도메인**과 1:1 관계를 맺으며, 학습 이력 추적을 위해 날짜별 스냅샷(History) 형태로 저장된다.
