# NextStep 프로젝트

## 1) 해결하려는 핵심 문제

### Problem A. 퇴근 후 “뭘 해야 할지” 결정이 안 나서 멍해짐

- '뭔가를 공부해야된다'라는 생각은 있음
- 근데 공부해야된다고 생각하는 주제가 너무 많음.
- 그래서 책상에 딱 앉았을 때, 뭘 해야될지 몰라서 시간이 낭비됨.

### Problem B. 공부를 해도 “기억이 안 남고 남는 게 없음”

- 이런저런 분야의 내용들을 잠깐씩 공부하고 기록하다보니 머릿속에 실력으로 쌓인다는 느낌이 없음.

### Problem C. “성장하고 있다”는 감각이 없음

- 주제별로 진행 과정이 구조화되어 있지 않음
- 오늘 한 일이 목표와 연결되어 보이지 않음

---

## 2) 앱의 한 문장 정의 (컨셉)

**“주제(Topic)별 현황을 기록하면, 앱을 켰을 때 오늘 해야 할 ‘가장 작은 next action’을 자동 추천하고, 실행 기록을 요약/복습 카드로 정리해 성장 흐름을 보여주는 학습 OS”**

---

## 3) 사용 시나리오 (너의 실제 생활 패턴)

- 퇴근 → 저녁 식사 → 자리 앉음
- 그날 공부/연습할 내용을 정하고 실행
- 최근에는 “무엇을 해야 할지” 길을 잃은 상태

👉 그래서 이 앱은 “기록 앱” 이전에 **공부하려고 책상에 앉았을 때, 사용자의 관심분야와 최근 학습 내역을 토대로 당일 학습 주제를 결정하는 앱**이 되어야 함.

## 5) MVP 기능(처음엔 딱 4개)

### (1) Topic 만들기 + 현황 기록

- Topic 이름, 목표 1줄, 현재/막힘 1줄, 우선순위(1~3)

### (2) 오늘의 추천(Today Plan)

- Topic별로 오늘 할 일 **1~3개** 자동 추천
- 추천은 “작고 구체적인 next action”(10~30분 단위, 산출물 존재)

### (3) 실행 기록(Log)

- 체크 + 한 줄 메모 + (선택) 링크
- **15초 컷**이 목표

### (4) 자동 요약/복습 카드(Review Card)

- 오늘 한 기록 → AI가 변환:

  - 오늘 배운 것 3줄
  - 내일 next action 1개
  - 헷갈린 개념 1개(짧은 설명)

---

## 6) 추천 엔진 설계: 룰 60% + AI 40% (혼합형)

### 룰 기반(결정 안정성)

- 최근 3일 이상 손 안 댄 Topic 우선
- 막힘 표시된 Topic → **난이도 낮춘 작업**으로 추천
- 가용 시간(20/45/90)에 맞춰 짧은 작업 우선
- 피곤 모드에서는 어려운 작업 배제

### AI 역할(표현/분해/요약)

- “Topic 현황/막힘”을 보고 **가장 작은 next action 문장 생성**
- 로그를 **요약 + 내일 행동**으로 정리

---

## 7) 홈 화면 UX(퇴근 후 최적화)

홈은 복잡하면 망함 → 들어가자마자 실행되게

1. 주제 선택
2. **Today Plan 자동 생성(3개 이하)**
3. **Start 버튼** → 타이머 + 체크/메모

핵심: “선택”이 아니라 **즉시 실행**.

# 계획(Gemini와 대화하면서 깨달은 내용들)

수정된 권장 프로세스
기존의 선형적인 구조보다는 아래와 같이 반복적이고 사용자 중심적인 구조로 수정하는 것을 제안합니다.

- 관찰 및 페인 포인트 기록: 3~5일간 해당 문제 발생 시 상황을 기록.
- 핵심 가치 정의: "이 앱은 [X]를 하기 위해 [Y]를 제공한다"라는 한 문장 정의.
- UI/UX 와이어프레임 (화면 설계): 손그림도 좋으니 사용 흐름을 먼저 확정.
- 기술 스택 선정 및 아키텍처 설계: 화면에 필요한 데이터를 기반으로 DB 및 API 설계.
- 핵심 기능 구현 (Walking Skeleton): 배포 환경까지 연결된 가장 기초적인 기능 구현.
- 실사용 및 반복 (Iterate): 직접 쓰면서 버그와 불편함을 수정.
